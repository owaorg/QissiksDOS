# ⚡ Qissiks | Async HTTP Stress Tool

**Qissiks** — это высокопроизводительная консольная утилита для тестирования сетевой нагрузки, написанная на Python с использованием асинхронного движка. Инструмент предназначен для проверки отказоустойчивости веб-ресурсов в рамках образовательных целей и нагрузочного тестирования.

## 🚀 Особенности

* **Полная асинхронность**: Построен на базе `asyncio` и `aiohttp`, что позволяет генерировать тысячи запросов без создания тяжелых системных потоков.
* **HEAD-оптимизация**: Использует HEAD-запросы для минимизации трафика и максимальной нагрузки на логику сервера.
* **Умное управление ресурсами**: Контроль конкурентности через `asyncio.Semaphore` предотвращает перегрузку локальной системы.
* **Cyber-Style UI**: Красивый консольный интерфейс с использованием библиотеки `Rich`.
* **Динамическая инжекция**: Система кастомизации через внешние JSON-конфиги для управления визуалом.

## 🛠 Технологический стек

* **Язык**: Python 3.12+
* **Сетевой уровень**: `aiohttp` (Asynchronous HTTP Client)
* **Интерфейс**: `Rich` (Terminal UI & Formatting)
* **Конфигурация**: `JSON`, `argparse`

pip install -r requirements.txt
⌨️ Использование
Запуск осуществляется через основной скрипт run.py. Вы можете настроить цель и уровень нагрузки через аргументы командной строки:


python run.py -u <URL_САЙТА> -t <КОЛИЧЕСТВО_ПОТОКОВ>
Пример:

python run.py -u https://example.com -t 500
Параметры:
-u, --url: Целевой URL для тестирования (обязательно).

-t, --threads: Лимит одновременных асинхронных задач (по умолчанию 100).

📂 Структура проекта
run.py — Точка входа в программу.

module/main.py — Обработка аргументов и запуск основного цикла.

module/dos_logic.py — Ядро асинхронной логики и обработка запросов.

module/interface.py — Генерация баннера и визуальных элементов.

module/injector.py — Работа с цветовыми схемами и конфигурациями.
